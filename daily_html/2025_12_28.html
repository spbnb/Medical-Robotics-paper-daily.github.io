See Less, See Right: Bi-directional Perceptual Shaping For Multimodal Reasoning

Abstract (English): Large vision-language models (VLMs) often benefit from intermediate visual cues, either injected via external tools or generated as latent visual tokens during reasoning, but these mechanisms still overlook fine-grained visual evidence (e.g., polylines in charts), generalize poorly across domains, and incur high inference-time cost. In this paper, we propose Bi-directional Perceptual Shaping (BiPS), which transforms question-conditioned masked views into bidirectional where-to-look signals that shape perception during training. BiPS first applies a KL-consistency constraint between the original image and an evidence-preserving view that keeps only question-relevant regions, encouraging coarse but complete coverage of supporting pixels. It then applies a KL-separation constraint between the original and an evidence-ablated view where critical pixels are masked so the image no longer supports the original answer, discouraging text-only shortcuts (i.e., answering from text alone) and enforcing fine-grained visual reliance. Across eight benchmarks, BiPS boosts Qwen2.5-VL-7B by 8.2% on average and shows strong out-of-domain generalization to unseen datasets and image types.

摘要（中文）: 大型视觉-语言模型(VLMs)通常受益于中间视觉线索，这些线索可以通过外部工具注入，或者在推理过程中生成为潜在的视觉标记，但这些机制仍然忽略了细粒度的视觉证据（例如，图表中的折线），在不同领域中的泛化能力较差，并且会带来高昂的推理时成本。在本文中，我们提出了双向感知塑造(BiPS)，它将问题条件下的掩码视图转换为双向的“看哪里”信号，从而在训练过程中塑造感知。BiPS首先在原始图像和保留证据的视图之间应用KL一致性约束，该视图仅保留与问题相关的区域，鼓励粗糙但完整的支持像素覆盖。然后，它在原始图像和证据消融视图之间应用KL分离约束，其中关键像素被掩盖，因此图像不再支持原始答案，从而抑制仅文本捷径（即，仅从文本中回答）并强制执行细粒度的视觉依赖。在八个基准测试中，BiPS平均将Qwen2.5-VL-7B提高了8.2%，并且在未见数据集和图像类型方面表现出强大的领域外泛化能力。

TLDR: This paper introduces Bi-directional Perceptual Shaping (BiPS), a training method for vision-language models that uses question-conditioned masked views to encourage fine-grained visual reasoning and prevent reliance on text-only shortcuts, achieving improved performance and generalization.

Agentic Structured Graph Traversal for Root Cause Analysis of Code-related Incidents in Cloud Applications

Abstract (English): Cloud incidents pose major operational challenges in production, with unresolved production cloud incidents cost on average over $2M per hour. Prior research identifies code- and configuration-related issues as the predominant category of root causes in cloud incidents. This paper introduces PRAXIS, an orchestrator that manages and deploys an agentic workflow for diagnosing code- and configuration-caused cloud incidents. PRAXIS employs an LLM-driven structured traversal over two types of graph: (1) a service dependency graph (SDG) that captures microservice-level dependencies; and (2) a hammock-block program dependence graph (PDG) that captures code-level dependencies for each microservice. Together, these graphs encode microservice- and code-level dependencies and the LLM acts as a traversal policy over these graphs, moving between services and code dependencies to localize and explain failures. Compared to state-of-the-art ReAct baselines, PRAXIS improves RCA accuracy by up to 3.1x while reducing token consumption by 3.8x. PRAXIS is demonstrated on a set of 30 comprehensive real-world incidents that is being compiled into an RCA benchmark.

摘要（中文）: 云事件给生产环境带来重大的运营挑战，未解决的生产云事件平均每小时造成超过200万美元的损失。先前的研究表明，代码和配置相关问题是云事件根本原因的主要类别。本文介绍了PRAXIS，一个用于管理和部署智能体式工作流的编排器，旨在诊断由代码和配置引起云事件。PRAXIS采用LLM驱动的结构化遍历，遍历两种类型的图：（1）服务依赖图（SDG），其捕获微服务级别的依赖关系；（2）吊床块程序依赖图（PDG），其捕获每个微服务的代码级别依赖关系。 这些图共同编码微服务和代码级别的依赖关系，而LLM作为这些图上的遍历策略运作，在服务和代码依赖关系之间移动，以定位和解释故障。与最先进的ReAct基线相比，PRAXIS将根本原因分析（RCA）的准确性提高了高达3.1倍，同时减少了3.8倍的token消耗。PRAXIS在一组30个全面的真实世界事件上进行了演示，这些事件正在被编译成一个RCA基准。

TLDR: PRAXIS uses an LLM-driven agentic workflow and structured graph traversal (SDG and PDG) to improve accuracy and reduce token consumption in root cause analysis (RCA) of code-related cloud incidents, demonstrating up to 3.1x accuracy improvement over ReAct. The approach is demonstrated on 30 real-world incidents.
